### Security Findings

#### [MODERATE] SEC-001: uuid Missing Buffer Bounds Check (GHSA-w5hq-g745-h8pq)

**Status:** ✅ Fixed (2026-04-23)

**ID:** SEC-001  
**Location:** `package-lock.json` / `node_modules/uuid` (transitive via `@opencode-ai/plugin` → `effect`)  
**Category:** Security  
**OWASP:** A03:2025  
**CWE:** CWE-787, CWE-1285  
**Effort:** easy

**Problem:**  
The `uuid` package (<14.0.0) contains a missing buffer bounds check in v3/v5/v6 when a `buf` argument is provided. This is reachable via the `effect` library, a dependency of `@opencode-ai/plugin`.

**Impact:**  
An attacker controlling the `buf` parameter could trigger an out-of-bounds write, potentially causing denial of service or memory corruption in downstream consumers.

**Remediation:**  
Update `@opencode-ai/plugin` to a version that resolves the transitive `uuid` dependency (≥1.4.3), or run `npm audit fix`.

#### [LOW] SEC-002: Unsanitized Path Resolution in loadMarkdownFile

**ID:** SEC-002  
**Location:** `packages/code-review/src/index.ts:8-13`  
**Category:** Security  
**OWASP:** A01:2025  
**CWE:** CWE-22  
**Effort:** easy

**Problem:**  
`loadMarkdownFile(name: string)` resolves paths using `path.resolve(moduleDirectory, name)` without validation. While currently only hardcoded constants are passed, future refactoring could accept dynamic input.

**Impact:**  
If ever called with user-controlled input, an attacker could supply `../../../etc/passwd` to read arbitrary files.

**Remediation:**  

```typescript
function loadMarkdownFile(name: string): string {
  const packagedPath = path.resolve(moduleDirectory, name)
  const sourcePath = path.resolve(moduleDirectory, "../src", name)
  const filePath = existsSync(packagedPath) ? packagedPath : sourcePath
  const baseDir = path.resolve(moduleDirectory, "..")
  if (!filePath.startsWith(baseDir)) {
    throw new Error("Invalid path: traversal detected")
  }
  return readFileSync(filePath, "utf8")
}
```

#### [LOW] SEC-003: Unvalidated File Path in /fix-report Command Template

**Status:** ✅ Fixed (2026-04-23)

**ID:** SEC-003  
**Location:** `packages/code-review/src/commands/fix-report.md:28`  
**Category:** Security  
**OWASP:** A01:2025  
**CWE:** CWE-22  
**Effort:** easy

**Problem:**  
The `fix-report` command template instructs the AI to read a file directly from the `$ARGUMENTS` placeholder without validating the path is within the intended project scope.

**Impact:**  
A malicious user invoking `/fix-report ../../../etc/passwd` could cause the AI agent to attempt reading sensitive files outside the project.

**Remediation:**  
Add an explicit validation step in the template before invoking the Read tool:

```markdown
1. Resolve the path from `$ARGUMENTS` relative to the project root.
2. Reject paths that contain `..` or resolve outside the project directory.
3. Use the Read tool to read the validated path.
```

### Architecture Findings

#### [HIGH] ARCH-001: Plugin Registration File Has Grown into a God Object (SRP Violation)

**Status:** ✅ Fixed (2026-04-23)

**ID:** ARCH-001  
**Location:** `packages/code-review/src/index.ts:15-157`  
**Category:** Architecture  
**Principle:** Single Responsibility Principle  
**Effort:** easy

**Problem:**  
`src/index.ts` declares 32 constants (16 prompts + 16 descriptions) and manually registers 16 config entries. The file has one reason to change for every new agent or command, violating SRP. At 157 lines, it is ~4× larger than Phase 1.

**Impact:**  
Adding a single new agent requires 3 separate edits. This is error-prone, difficult to review, and increases merge conflict risk.

**Remediation:**  
Extract metadata into typed arrays and iterate:

```typescript
const AGENTS: { name: string; description: string; path: string }[] = [
  { name: "security-auditor", description: "...", path: "agents/security-auditor.md" },
  // ...
]

const COMMANDS: { name: string; description: string; path: string }[] = [
  { name: "review", description: "...", path: "commands/review.md" },
  // ...
]

export const AppVerkCodeReviewPlugin: Plugin = async () => ({
  config: async (config) => {
    config.agent ??= {}
    for (const a of AGENTS) {
      config.agent[a.name] = { description: a.description, prompt: loadMarkdownFile(a.path) }
    }
    config.command ??= {}
    for (const c of COMMANDS) {
      config.command[c.name] = { description: c.description, template: loadMarkdownFile(c.path) }
    }
  },
})
```

#### [HIGH] ARCH-002: Plugin Not Closed for Modification (OCP Violation)

**ID:** ARCH-002  
**Location:** `packages/code-review/src/index.ts:80-155`  
**Category:** Architecture  
**Principle:** Open/Closed Principle  
**Effort:** easy

**Problem:**  
The plugin factory manually assigns each agent and command. There is no extension mechanism — adding any new capability requires modifying `index.ts` in three places.

**Impact:**  
Discourages incremental additions and prevents consumers from injecting custom agents without forking.

**Remediation:**  
Apply the same data-driven approach as ARCH-001. Optionally, export the registry arrays so external packages can append entries before plugin initialization.

#### [MEDIUM] MAINT-003: Fragile File Resolution with Hardcoded Relative Path Fallback

**Status:** ✅ Fixed (2026-04-23)

**ID:** MAINT-003  
**Location:** `packages/code-review/src/index.ts:8-13`  
**Category:** Maintainability  
**Effort:** easy

**Problem:**  
`loadMarkdownFile` uses `path.resolve(moduleDirectory, "../src", name)` as a fallback. This assumes `src/` is always a sibling to the execution directory, which breaks if the package is bundled, symlinked, or installed from a git submodule.

**Remediation:**  
Remove the fallback (since `copy-assets.mjs` guarantees assets exist in `dist/`) or resolve from the package root. Fail fast with a descriptive error instead of silent fallback.

#### [MEDIUM] MAINT-004: No Error Handling When Markdown File Is Missing

**ID:** MAINT-004  
**Location:** `packages/code-review/src/index.ts:12`  
**Category:** Maintainability  
**Effort:** trivial

**Problem:**  
`readFileSync(filePath, "utf8")` is called without verifying `filePath` exists. If both paths are missing, Node throws a generic `ENOENT` with no context about which agent/command failed.

**Remediation:**  

```typescript
function loadMarkdownFile(name: string): string {
  const packagedPath = path.resolve(moduleDirectory, name)
  const sourcePath = path.resolve(moduleDirectory, "../src", name)
  const filePath = existsSync(packagedPath) ? packagedPath : sourcePath
  if (!existsSync(filePath)) {
    throw new Error(
      `Markdown asset not found for "${name}". Tried: ${packagedPath} and ${sourcePath}`
    )
  }
  return readFileSync(filePath, "utf8")
}
```
