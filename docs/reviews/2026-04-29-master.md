## Security Findings

### [LOW] SEC-001: grep commands in SKILL.md lack directory exclusions

**Status:** ✅ Fixed (2026-04-29)

**ID:** SEC-001  
**Location:** `packages/code-review/src/skills/standards-discovery/SKILL.md`  
**Category:** Security  
**Effort:** trivial

**Problem:**  
The `grep -rni` commands in the skill file do not exclude `.git/`, `node_modules/`, or `.venv/` directories. While this is not a security vulnerability (the commands are hardcoded templates for AI agents, not runtime code processing user input), it may cause agents to search unnecessary files in projects with large `node_modules` directories.

**Impact:**  
Performance overhead during AI agent discovery workflows. No direct security risk.

**Remediation:**  
Add `--exclude-dir` flags to `grep` commands for consistency with the `find` command exclusions:

```bash
# Before
grep -rni "naming convention\|camelCase\|snake_case\|PascalCase\|kebab-case\|UPPER_CASE" --include="*.md" . 2>/dev/null

# After
grep -rni "naming convention\|camelCase\|snake_case\|PascalCase\|kebab-case\|UPPER_CASE" --include="*.md" --exclude-dir={.git,node_modules,.venv,vendor} . 2>/dev/null
```

## Architecture Findings

### [HIGH] ARCH-001: Self-Referential Circular Dependency with Stale Version Tag

**Status:** ✅ Fixed (2026-04-29)

**ID:** ARCH-001  
**Location:** `package.json:22`  
**Category:** Architecture  
**Effort:** trivial

**Problem:**  
The root `package.json` (version `0.2.13`) declares a dependency on an older version of itself:

```json
"dependencies": {
  "@opencode-ai/plugin": "^1.14.19",
  "av-opencode-plugins": "github:AppVerk/av-opencode-plugins#v0.2.12"
}
```

This creates a circular dependency. The hardcoded tag `v0.2.12` became stale immediately upon bumping to `0.2.13`. Package managers may install the older version into `node_modules`, causing duplicate package issues and confusing resolution.

**Impact:**  
- Package resolution conflicts and duplicate installations  
- Manual version maintenance required on every release  
- Fundamental dependency hygiene violation (a package should not depend on itself)

**Remediation:**  
Remove the self-referential dependency entirely:

```json
// After
"dependencies": {
  "@opencode-ai/plugin": "^1.14.19"
}
```

The `.opencode/opencode.json` already references the local path for OpenCode plugin resolution, making this dependency redundant.

## Maintainability Findings

### [HIGH] MAINT-001: Incomplete Standards-Discovery and Pre-Analysis Rollout

**Status:** ✅ Fixed (2026-04-29)

**ID:** MAINT-001  
**Location:** `packages/code-review/src/agents/*.md`, `packages/code-review/src/commands/*.md`, `packages/code-review/tests/build-output.test.ts`  
**Category:** Maintainability  
**Effort:** medium

**Problem:**  
A new global skill (`standards-discovery`) and its supporting Pre-Analysis infrastructure were partially implemented across the codebase with multiple maintainability defects:

1. **DRY Violation (18 copies):** The same 11-line Pre-Analysis block was copy-pasted into 9 source markdown files and 9 corresponding `dist/` build artifacts:
   - `src/agents/{code-quality-auditor,documentation-auditor,feedback-analyzer,fix-auto,security-auditor}.md`
   - `src/commands/{analyze-feedback,fix,fix-report,review}.md`
   - Plus all 9 `dist/` counterparts

2. **Inconsistent Agent Coverage (7 of 16 missing):** The Pre-Analysis step was not added to:
   - `cross-verifier.md`
   - `challenger.md`
   - `skill-secret-scanner.md`
   - `skill-sast-analyzer.md`
   - `skill-dependency-scanner.md`
   - `skill-architecture-analyzer.md`
   - `skill-linter-integrator.md`

3. **Minimal Test Coverage:** Only 1 smoke test line added (`build-output.test.ts` verifying file existence). No tests for skill content, registry integration, or Pre-Analysis injection.

4. **Documentation Gaps:** `AGENTS.md`, `docs/plugins/skill-registry.md`, and `docs/plugins/code-review.md` were not fully updated to reflect the new skill.

**Impact:**  
- Any Pre-Analysis update requires editing 18 files with high drift risk  
- The feature is effectively undiscoverable by users  
- Inconsistent behavior across the code review suite  
- No regression protection

**Remediation:**  
1. **Centralize Pre-Analysis injection** in `packages/code-review/src/index.ts` instead of copy-pasting into markdown files:

```typescript
const PRE_ANALYSIS_BLOCK = `## Pre-Analysis Step: Discover Project Standards
...`;

function injectPreAnalysis(content: string): string {
  const match = content.match(/^---\n[\s\S]*?\n---\n/)
  if (!match) return content
  const endIndex = match[0].length
  return content.slice(0, endIndex) + PRE_ANALYSIS_BLOCK + content.slice(endIndex)
}
```

2. **Apply uniformly** to all agents or document explicit exclusion criteria  
3. **Add targeted tests** for skill content and registry integration  
4. **Update all documentation** (`AGENTS.md`, `skill-registry.md`, `code-review.md`)
